12.1. Usar el ejemplo Blog del GitHub

Para demostrar el uso de Doctrine ORM, en este capítulo crearemos un Blog realista. Sus principales características son las siguientes:

  • Se guardarán las publicaciones del blog en la base de datos y se provee de una interfaz de usuario para ver y gestionar las publicaciones.
  • Suponemos que el blog tiene un solo autor por cada autor, mientras que los comentarios pueden ser agregados por muchos lectores del blog.
  • La sitio web tiene dos páginas: la página Home y la página Admin. La primera mostrará la lista de las publicaciones recientes, mientras que la otra página permitirá agregar, editar, ver y borrar las publicaciones.

Para ver las pantallas de ejemplo del sitio web podemos ver la figura 12.1 y 12.2 más abajo:

Figure 12.1. Blog home page

Figure 12.2. Blog admin page

Para descargar la aplicación Blog podemos visitar esta página y hacer clic en el botón Clone or Download para descargar el código en un archivo ZIP. Cuando la descarga esta lista desempaquetamos el archivo en algún directorio.

Luego navegamos hasta el directorio blog que contiene el código fuente de la aplicación Blog.

/using-zf3-book-samples
  /blog
  ...

El Blog es un sitio web simple que podemos instalar en nuestra computadora. Para instalar este ejemplo podemos o editar nuestro sitio virtual por defecto o crear un nuevo sitio virtual de Apache. Después de editar el archivo se debe reiniciar el servidor HTTP Apache y abrir el sitio web en nuestro navegador web.

Para ver las instrucciones de instalación detalladas podemos revisar el archivo README.md ubicado dentro del directorio blog.

Para que el Blog de ejemplo funcione es necesario crear una base de datos MySQL. Las instrucciones sobre como hacer esto se encuentran en la siguiente sección.
